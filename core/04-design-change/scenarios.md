# 시나리오 테스트

## 시나리오 A: 단일 요소 수정 (경량)

**상황**: "로그인 버튼 색상 파란색으로 바꿔줘"

**올바른 진행**:
1. 경량 절차 적용
2. `templates/login.html` 확인
3. "[디자인 수정] login.html의 로그인 버튼 색상 → 파란색으로 수정합니다." 보고 후 바로 구현

**잘못된 진행**: screen-domain.md 읽고 → 계획 수립 → 승인 요청 (과잉 절차)

## 시나리오 B: 페이지 단위 변경 (표준)

**상황**: "감정 분석 페이지 워드클라우드 옵션 추가해줘"

**올바른 진행**:
1. 표준 절차 적용
2. screen-domain.md에서 분석 페이지 파일 확인: `web/templates/index.html`
3. batch_processor.py에서 워드클라우드 옵션 분석
4. 변경 계획 수립 후 사용자 승인 → 구현

**검토 결과**: ✅ 규모 판단으로 불필요한 절차 제거 가능
