# 계획서 표준 템플릿 사용 안내

> 본 폴더의 템플릿은 [`03.plan-mode.md`](../03.plan-mode.md) §계획서 표준 템플릿 참조 규칙에 따라 계획서 작성 시 반드시 참조한다.

---

## 템플릿 선택 기준

계획서 작성 전 아래 분류표에서 작업 유형을 확인하고, 해당 템플릿 파일을 연다.

| 작업 유형 | 템플릿 파일 | 판단 기준 |
|-----------|-------------|-----------|
| **A. 버그 수정/핫픽스** | `type-a-bugfix.md` | 기존 기능이 잘못 동작함. 원인 추적 + 최소 수정 |
| **B. 기능 개선/신규 기능** | `type-b-feature.md` | 새 요구사항/기능 추가. 기존 코드 확장 |
| **C. 설계/아키텍처/데이터셋** | `type-c-design.md` | 구조/스키마/데이터 설계. 결정 필요 사항 다수 |
| **D. 리팩토링/성능 개선** | `type-d-refactor.md` | 코드 품질/속도/메모리 개선. 동작 변경 없음 |
| **E. DB 마이그레이션** | `type-e-migration.md` | DB 스키마 변경/데이터 이관. 이전 호환 필요 |
| **F. 복구/롤백** | `type-f-recovery.md` | 실수로 제거된 기능 복원. 이전 상태로 되돌림 |

### 혼합 유형 처리

하나의 계획서가 여러 유형에 걸칠 경우, **주 유형 1개**를 선택하고 다른 유형의 필수 섹션을 추가로 포함한다.

---

## 공통 메타 블록 (모든 계획서 최상단 필수)

```markdown
# 계획서 — {작업명}

> 상태: PND | 작성일: YYYY-MM-DD
> 작업 유형: {A|B|C|D|E|F}
> 선행: (관련 계획서가 있을 경우 경로)
```

---

## 공통 섹션 (모든 유형 필수 포함)

모든 계획서는 아래 섹션을 반드시 포함해야 한다. 섹션 순서는 템플릿에 명시된 순서를 따른다.

| 섹션 | 설명 |
|------|------|
| `## 수정 이력` | 계획서 변경 이력 (03.plan-mode.md §7 형식 준수) |
| `## 1. 배경 및 목적` | 왜 이 작업이 필요한가 |
| `## N. 구현/수정 상세` | 구체적인 변경 내용 |
| `## N+1. 영향도 분석` | 변경 파일 목록 + 영향 범위 |
| `## N+2. 테스트/검증 계획` | 시나리오 기반 검증 |
| `## N+3. 리스크 및 제약` | 알려진 위험/한계 |
