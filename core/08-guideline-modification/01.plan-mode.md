# 01. Plan Mode 행동 규칙

## ⛔ 최우선 잠금 규칙

**Plan mode에서 파일 수정/생성/삭제 등 실행 행동을 하는 실수가 반복 발생.**

### Plan Mode에서 할 수 있는 것 (읽기)
- 파일 읽기 (`read_file`)
- 파일 검색 (`glob`, `grep`)
- 경로 분석
- 계획 수립

### Plan Mode에서 할 수 없는 것 (실행) ❌
- 파일 수정/생성/삭제
- 코드 작성/실행
- bash 명령어 실행
- git commit

### 올바른 진행
```
1. Plan 수립
2. 사용자 확인 요청: "Plan准备好了. 실행할까요?"
3. 사용자가 "수행", "실행" 등 명시적 승인
4. Build mode 전환 후 실제 작업 수행
```

### 실수 방지 체크리스트
- [ ] Plan을 수립했는가?
- [ ] 사용자에게 실행 승인을 요청했는가?
- [ ] 사용자가 "수행"이라고 명시적으로 말했는가?
- [ ] 언급하는 함수/클래스/변수가 실제 코드베이스에 존재하는가? (grep/read 확인 완료)
- [ ] DB 경로, 설정 파일 등 인프라 정보가 실제 코드와 일치하는가?
- [ ] 추측(predict)한 내용이 없는가? (모든 기술은 확인된 사실인가?)
