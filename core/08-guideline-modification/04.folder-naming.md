# 04. 폴더 명칭 규칙

> 문서를 분리할 때 폴더 명칭은 **나침반의 역할을 하는 문서의 이름**을 따름.

## 올바른 예시

**분리 전:**
```
docs/msys/
├── core-admin-page-rules.md    ❌ 파일명에 접두사 사용
└── core-design-change-workflow.md
```

**분리 후 (올바름):**
```
docs/msys/
└── core/
    ├── admin-page-rules.md
    └── design-change-workflow.md
core.md                         ← 나침반 (참조만)
```

### 중첩 구조 (나침반 문서가 하위 폴더에 있을 때)

나침반 문서가 이미 하위 폴더에 있는 경우, 분리 폴더도 **동일한 하위 폴더 안에 생성**한다.

**분리 전:**
```
.clinerules/core/00-core/
    ├── 01.global-rules.md
    ├── 02.triggers.md
    └── 03.plan-mode.md          ← 116줄, 내용 증가로 분리 필요
```

**분리 후 (올바름):**
```
.clinerules/core/00-core/
    ├── 01.global-rules.md
    ├── 02.triggers.md
    ├── 03.plan-mode.md          ← 나침반 (참조만)
    └── 03-plan-mode/            ← 동일 디렉토리(00-core/)에 생성
        ├── README.md
        ├── type-a-bugfix.md
        └── ...
```

## 핵심 원칙

1. **나침반 문서명 = 폴더명**: 분리된 문서를 담는 폴더는 나침반 문서의 이름과 같음
2. **소문자 사용**: 폴더명과 파일명은 소문자 사용
3. **케밥 케이스**: 단어 구분은 하이픈(`-`) 사용

## 폴더 생성 절차

1. 나침반 문서명 확인 (예: `core.md`)
2. 나침반 문서명에서 `.md` 제거 → 폴더명 (예: `core/`)
3. **폴더는 나침반 문서와 동일한 디렉토리에 생성한다.**
   - 나침반 문서가 `A/B/core.md`에 있으면 → 폴더는 `A/B/core/`에 생성
   - 나침반 문서가 `A/B/03.plan-mode.md`에 있으면 → 폴더는 `A/B/03-plan-mode/`에 생성
4. 분리된 문서들을 해당 폴더로 이동
5. 나침본 문서에 하위 문서 목록 추가

## 잘못된 예시 (⚠️ 주의)

### 평탄 구조에서의 실수

```
docs/msys/
├── msys-core/           ❌ msys 접두사 불필요
│   └── admin-page-rules.md
└── msys-design/        ❌ msys 접두사 불필요
    └── design-change-workflow.md
```

### 중첩 구조에서의 실수 (상위 디렉토리에 생성)

```
.clinerules/core/
    ├── 00-core/
    │   ├── 01.global-rules.md
    │   ├── 02.triggers.md
    │   └── 03.plan-mode.md    ← 나침반
    └── 03-plan-mode/          ❌ 동일 디렉토리(00-core/)가 아닌 상위(core/)에 생성하면 안 됨
        └── ...
```

> **핵심**: 폴더 명칭은 나침반 문서명에서 `.md`를 제거한 것. 프로젝트 접두사는 불필요.
> **중요**: 폴더는 나침반 문서와 **정확히 동일한 디렉토리**에 생성해야 한다. 나침반 문서가 하위 폴더에 있으면 분리 폴더도 그 하위 폴더 안에 생성한다.
