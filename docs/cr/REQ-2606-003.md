### 변경 요약 (1줄, 50자 이내)
> 갤러리 다운로드 500 에러 수정 (datetime 미임포트 + Windows 파일 잠금)

### 변경 배경 / 이유
> `api_deploy_gallery_download()` 구현 시 `datetime` 모듈이 임포트되지 않아 `NameError` 발생. Flask 커스텀 500 핸들러가 JSON을 반환하는 개발환경에서는 "다운로드 실패: Internal Server Error", nginx 프록시가 HTML 에러 페이지를 반환하는 내부망에서는 "Unexpected token '<', <!doctype..." 에러 메시지로 표출됨. 추가로 Windows 환경에서 `NamedTemporaryFile` 파일 핸들이 열린 채로 동일 파일을 재오픈하면 `PermissionError`가 발생하는 구조적 문제도 동반 수정.

### 변경 전 → 변경 후

- Before:
  - `datetime.now()` 사용 (미임포트)
  - `NamedTemporaryFile` 핸들 열린 채 `ZipFile(tmp.name, 'w', ...)` 시도
  - `executeDownload()` 에러 응답이 HTML이면 `res.json()` 자체가 SyntaxError 발생
  - `api_deploy_gallery_download()` 전체에 try/except 없음

- After:
  - `from datetime import datetime as _dt` 로컬 임포트 추가
  - `tmp.close()` 후 `tmp_name` 변수로 ZipFile 오픈
  - `res.json()` 실패 시 `HTTP {status}` 폴백 처리
  - 함수 전체 `try/except` 감싸기

### 영향 범위
> `deploy_gallery.html` 갤러리 다운로드 기능, `api_deploy_session_download()` (동일 버그 선제 수정)

```python
# Before — perspective_routes.py
tmp = tempfile.NamedTemporaryFile(delete=False, suffix='.zip')
with zipfile.ZipFile(tmp.name, 'w', zipfile.ZIP_DEFLATED) as zf:
    ...

                 download_name=f'gallery_{datetime.now().strftime(...)}.zip')  # NameError

# After
from datetime import datetime as _dt
try:
    ...
    tmp = tempfile.NamedTemporaryFile(delete=False, suffix='.zip')
    tmp_name = tmp.name
    tmp.close()  # Windows: 핸들 닫고 이름으로 재오픈
    with zipfile.ZipFile(tmp_name, 'w', zipfile.ZIP_DEFLATED) as zf:
        ...
    return send_file(tmp_name, ...,
                     download_name=f'gallery_{_dt.now().strftime(...)}.zip')
except Exception as e:
    return jsonify({'success': False, 'error': str(e)}), 500
```
