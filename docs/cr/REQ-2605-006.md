### 변경 요약 (1줄, 50자 이내)
> 그룹 카드/대시보드 total 계산에서 상태코드 필터 제거 및 SQL 그룹핑 수정

### 변경 배경 / 이유
- 수집 스케줄 그룹 카드의 total이 CD905(재수집), CD906(지연) 항목을 제외하고 계산되어 실제 cron 실행 횟수보다 적게 표시됨
- 대시보드의 총 호출 건수와 기타(no_data) 카운트가 CD905/CD906/CD908을 누락하여 불일치 발생
- data_spec 모듈의 template literal truncation으로 인한 JS 문법 오류

### 변경 전 → 변경 후
- Before: 그룹 카드 total = `validJobs.length` (CD901,902,907,908만 필터)
- After : 그룹 카드 total = `allSubGroupJobs.length` (모든 상태코드 포함)
- Before: SQL no_data = CD903만 카운트
- After : SQL no_data = CD903+CD905+CD906+CD908 그룹핑
- Before: 성공률 = 성공/(성공+실패+기타)
- After : 성공률 = 성공/(성공+실패) — 기타 제외

### 영향 범위
- 수집 스케줄 페이지의 그룹 카드 및 팝업 카운트
- 대시보드 페이지의 총 호출 건수, 기타 카운트, 성공률
- data_spec 페이지 JS 문법 오류 수정
