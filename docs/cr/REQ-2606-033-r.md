### 변경 사항 요약
> 배치 처리 시 문장 단위 KoTE 원시 점수를 캐시로 저장하고, 그룹 분석·제출용 저장 조회 시 캐시를 재사용하여 KoTE 재실행을 완전 제거합니다.

### 상세 변경 사항

**1. 배치 처리 — 문장 단위 KoTE 캐시 생성**
- 메타데이터 생성 시 문서를 문장으로 분할 후 각 문장의 KoTE 원시 점수(pos/neg/neutral)를 `sentence_emotion_cache` 필드에 저장
- DB 스키마 변경 없음 (`evaluations.data` JSON 필드 추가만)

**2. 그룹 분석 / 제출용 저장 — 캐시 재사용**
- 4개 호출 지점 모두 캐시를 우선 사용 → KoTE 재실행 없음
- 반전 표지어 규칙 · 사용자 교정은 조회 시점 동적 적용 유지
- 기존 배치(캐시 없음)는 공유 헬퍼 fallback으로 기존과 동일하게 동작

**3. 공유 헬퍼 모듈 신설 — `sentence_emotion.py`**
- `compute_sentence_raw_scores(doc)`: 배치 캐시 생성과 그룹 분석 fallback이 공유하는 경량 헬퍼
- 영어 문장 감지(70% 초과) → 욕설 필터 분기 유지

**4. `split_sentences` 의존 구조 개선**
- 정의를 `perspective_service.py`(matplotlib 포함)에서 `text_preprocessing.py`(경량)로 이전
- 배치 워커 경로에서 matplotlib 의존 완전 차단

**2. 영향받는 페이지**
- 그룹 분석 화면
- 제출용 저장 (save_to_deploy)

### Known Issues
- 기존 배치 데이터는 캐시 없음 → fallback(즉석 KoTE)으로 동작. 재배치 후 캐시 이득 발생.
- 측정 표본 문서가 짧아 평균 1.30문장/문서. 실제 평가문이 길면 배수 증가(배치 1회성).
