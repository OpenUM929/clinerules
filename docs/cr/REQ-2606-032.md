### 변경 요약 (1줄, 50자 이내)
> 그룹분석·제출용저장 회귀 수정 (target_employee_id 복원)

### 변경 배경 / 이유
> REQ-2606-030(비속어 가명 복원) 커밋이 load_all_batches()의 target_employee_id를 가명 ID(emp_id)→실명(display_name)으로 바꾸는 부작용 발생. 이로 인해 매칭 함수(_get_employee_metadata, _get_evaluations_for_employee)가 전원 불일치→빈 결과를 반환하여 제출용 저장·그룹분석 워드클라우드가 동작하지 않는 회귀 발생.

### 변경 전 → 변경 후
- Before: `target_employee_id = display_name` (실명, 02534c8 도입) — 매칭 실패, 워드클라우드 생성 불가
- After : `target_employee_id = emp_id` (가명 ID) — 매칭 정상, 상위 enrich/real 계층에서 실명 복원

### 영향 범위
> perspective_service.py의 load_all_batches() 단일 함수. 비속어 가명 복원(REQ-2606-030)은 별도 위치(_generate_profanity_cell, build_profanity_summary)에서 처리하므로 영향 없음.
