### 변경 요약 (1줄, 50자 이내)
> 배치 시 문장 단위 KoTE 캐시 저장 + 그룹 분석 재사용

### 변경 배경 / 이유
> 배치 처리에서 analyze_emotion(doc)으로 문서 단위 KoTE를 실행·저장하고 있었으나, 그룹 분석·제출용 저장은 이 결과를 전혀 활용하지 않고 매 조회마다 split_sentences + analyze_emotion(sent) × N문장을 재실행했다. 직원 1명 조회 시 해당 직원 모든 평가의 문장 수만큼 KoTE가 반복 실행되는 설계 모순을 해소한다. 배치 처리 1회 비용(+16.4ms/평가, 1.9만명 기준 +약 26분)으로 매 조회 KoTE 재실행을 완전 제거한다.

### 변경 전 → 변경 후
- Before: 그룹 분석/제출용 저장 조회마다 split_sentences(doc) + analyze_emotion(sent)×N 재실행. 배치 저장 결과 미활용.
- After : 배치 시 sentence_emotion_cache(문장별 pos/neg/neutral) 저장 → 조회 시 캐시 읽기(KoTE 재실행 없음). 반전 규칙·사용자 교정은 조회 시점 동적 적용 유지.

### 영향 범위
> - 배치 처리(create_employee_metadata): KoTE 1회 추가(문장 단위). 신규 배치부터 캐시 생성됨.
> - 그룹 분석/제출용 저장: 신규 배치는 캐시 사용(KoTE 0회). 기존 배치는 fallback(KoTE 재실행, 기존과 동일).
> - DB 스키마 변경 없음. evaluations.data JSON에 sentence_emotion_cache 필드 추가만.
> - split_sentences 사용처(perspective_routes, test_routes 등) 영향 없음(재노출 유지).
