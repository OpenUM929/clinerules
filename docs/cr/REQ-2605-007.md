### 변경 요약 (1줄, 50자 이내)
> Nav 메뉴를 `common.css`의 `.container`에서 분리하여 크기 고정

### 변경 배경 / 이유
- 관리자 설정(`/mngr_sett`) 메뉴 선택 시 Nav 메뉴의 너비가 1900px로 확장되고 높이도 함께 변하는 현상 발생
- 이전 AI가 `.container`를 `#mngr_sett_page.container`로 스코핑했으나, 콘텐츠 overflow로 인한 `<body>` 너비 확장이 Nav의 `bg-blue-800` 배경 영역과 `.container`에 영향을 주는 것이 근본 원인
- Nav를 `common.css`의 `.container` 클래스에서 완전히 분리하여 어떤 페이지의 CSS 재정의에도 영향을 받지 않도록 함

### 변경 전 → 변경 후
- Before: `navbar.html`이 `class="container"` 사용 → `mngr_sett`의 `page-specific-style`의 영향을 받아 너비/높이 변동
- After : `navbar.html`이 `class="nav-container"` 사용 → 독립 클래스로 완전히 격리되어 크기 고정

### 영향 범위
- **Navbar**: `nav-container` 클래스 적용으로 항상 1600px 고정
- **다른 메뉴**: 기존 `.container` 그대로 유지, 영향 없음
- **mngr_sett**: `mngr-sett-container` 사용으로 1920px 확장 기능 유지
